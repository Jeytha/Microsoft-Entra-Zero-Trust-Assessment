#  What Is Zero Trust?

**Zero Trust** is a modern security framework built on three core principles:

1. **Verify explicitly**  
   Always authenticate & authorize based on all available signals (identity, device, risk, location).

2. **Use least privilege access**  
   Give users the minimum rights required—no broad or permanent permissions.

3. **Assume breach**  
   Architect systems as if an attacker is already inside; minimize blast radius.

Microsoft applies Zero Trust across:
- Identity & Access  
- Devices  
- Applications  
- Networks  
- Infrastructure  
- Data  
- Monitoring  

---

#  What Is the Microsoft Entra Zero Trust Assessment?

The **Microsoft Entra Zero Trust Assessment** is an automated PowerShell-based tool that analyzes your tenant’s security posture and provides Microsoft-aligned recommendations.

It evaluates:

###  **Identity Security**
- MFA usage  
- Conditional Access  
- Privileged roles  
- Risk-based policies  
- Password protection  

###  **Device Security**
- Intune enrollment  
- Device compliance  
- Platform health  
- Secure baselines  

###  **Configuration Security**
- Endpoint protection policies  
- Defender posture  
- Intune configuration profiles  
- Compliance baselines  

The result is a multi-section HTML report summarizing risks, gaps, and Zero Trust readiness.

---

#  Technical Background

The assessment tool uses:

- **Microsoft Graph API**  
  Queries identity, authentication, CA, roles, user risk, and sign-in data.

- **Azure Resource Manager (ARM)**  
  Reads configuration, device management, and Intune policy data.

- **ZeroTrustAssessment PowerShell Module**  
  A Microsoft-built module that:
  - Authenticates to Microsoft Graph  
  - Connects to Azure  
  - Runs ~200 security checks  
  - Produces a full HTML report  

The tool is **read-only** and **makes no configuration changes**.

#  Prerequisites

To run the assessment, the environment requires:

### ✔ PowerShell 7.x  
Windows PowerShell 5.1 is not supported.

### ✔ Modules Installed
```powershell
Install-Module Microsoft.Graph -Scope CurrentUser
Install-Module Az -Scope CurrentUser
Install-Module ZeroTrustAssessment -Scope CurrentUser
```

###  Permissions Needed
A user with:
- **Global Reader** (minimum)  
- **Global Administrator** (for granting Graph consent)

###  An Entra ID Tenant  
Assessment is performed against a **test tenant** created for this portfolio.

#  How the Assessment Was Performed

### 1️ Installed required modules  
### 2️ Created a fresh test tenant (empty, default configuration)  
### 3️ Created an Entra ID user with Global Admin + Global Reader  
### 4️ Connected to Microsoft Graph  
```powershell
Connect-MgGraph -Scopes "Directory.Read.All"
```

### 5️ Connected to Azure  
```powershell
Connect-AzAccount -Tenant <TenantID>
```

### 6️ Ran the assessment  
```powershell
Connect-ZtAssessment
Invoke-ZtAssessment -Path "C:\ZTReport"
```

### 7️ Exported reports as PDFs  
### 8️ Redacted all sensitive data (tenant IDs, domains, emails)  
### 9️ Uploaded only safe files to this repository

#  Executive Summary of Findings

The assessment was run on a **fresh, unconfigured test tenant**.  
The results accurately reflect what Zero Trust tooling identifies in an environment with **zero security controls applied**.

###  Identity Findings
- MFA not enforced  
- No Conditional Access policies  
- No Identity Protection configuration  
- No privileged access baselines  
- No authentication hardening  
- Many High / Medium identity risks  

###  Device Findings
- No devices enrolled  
- No compliance policies  
- No Defender baselines  
- All device posture tests marked "Skipped" or "Not Configured"  

###  Configuration Findings
- No Intune profiles or baselines  
- No endpoint security configuration  
- No RBAC partitioning  
- No data protection or governance controls  

###  Overall Risk Posture  
**This tenant represents a perfect "before" picture**, ideal for showcasing:

- Zero Trust assessment skills  
- Interpretation of cloud security risks  
- Understanding of Microsoft Entra & Intune posture  
- Technical documentation ability  
- Cloud hardening fundamentals

#  Analysis of Results

###  **Why Identity Failed**
A default tenant has:  
- No MFA  
- No CA  
- No admin protection  
- No password hashing rules  
- No sign-in risk policies  

This exposes organizations to:  
- Credential compromise  
- Token theft  
- Phishing & MFA fatigue  
- Lateral movement  

###  **Why Devices Scored Low**
Devices are secure **only when managed**.  
With no enrollment:  
- Compliance = 0  
- Baselines = 0  
- Posture visibility = none  

###  **Why Configuration Scored Low**
Microsoft Defender + Intune settings form the backbone of Zero Trust device security.  
A fresh tenant has none applied.

###  **Expected? Yes. Valuable? Absolutely.**
This empty tenant provides the **ideal demonstration** for how Zero Trust tooling behaves in an unsecured environment—perfect for interviews and portfolio demonstration.
